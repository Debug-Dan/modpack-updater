Modpack Launcher

Q: Is this a virus?

A: This launcher is not a virus. I know that .bat files can typically be associated with viruses, trojans, spam, etc. I have a lot of experience in making batch files and honestly it was easier to make an extremely lightweight launcher out of what is essentially command prompt code versus making an entire .exe program.

For the purpose of transparency, this launcher does download files from the internet, and I know that can come across as extremely scary because theoretically the launcher could download anything it wanted. Because of this, the launcher could theoretically not contain any harmful code right now and then at a later date be updated to have harmful code. My counter to this claim is that basically any program you have installed on your computer could do the same and this file is no different.

Here is a detailed list of what this launcher will download from the internet:

  1. An updated version of itself. This is required to add new options to the troubleshooter, new features, and also to add updated download links to newer versions of the modpacks. Because many modpacks in the launcher are stored on Google Drive, each update to the modpack is a new download link and therefore must be updated.
  2. Prism Launcher & its dependancies. This launcher piggybacks on Prism Launcher to launch modpack instances and therefore has an automated setup for installing it.
  3. Modpack files. More specifically, it downloads a 7-zip self-extracting archive. A 'self-extracting archive' is a feature of 7-zip that is basically just a .zip/.7z archive file but in the .exe format so it can extract itself.

Here is a detailed list of what files this launcher has access to on your local machine:

  1. The folder you place 'modpack-launcher.bat'. It downloads files here and creates a new folder called 'modpack-launcher-files' for storing dependancies for the launcher.
  2. '%localappdata%/Programs/PrismLauncher'. This is the Prism Launcher install directory. The launcher does this to launch modpack instances.
  3. '%appdata%/PrismLauncher/instances'. This is the location where Prism Launcher has the instances. The launcher does this to install/update/delete instances

I hope this helped to answer your question :)
